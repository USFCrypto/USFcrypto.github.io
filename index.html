Why this starting point works

Go live first, refine second.

This starter is deliberately small. Once it is live, you can keep improving content, navigation, and page structure without waiting for the full migration.

  • Simple static files that GitHub Pages can serve immediately
  • Easy to port later into al-folio or another Jekyll structure
  • Uses your real visual assets from the current site

Recommended first live version

Publish just four pages:

  1. Home
  2. Research
  3. People
  4. Programs

That is enough to establish the new website and iterate in public.
